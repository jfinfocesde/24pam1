# Semana 4

## Métodos en POO Java

Los métodos son bloques de código reutilizable que realizan tareas específicas dentro de una clase en la programación orientada a objetos (POO) de Java. Son esenciales para la modularidad, encapsulación y reutilización del código.

**Elementos clave de un método:**

* **Nombre:** Identifica el método de forma única.
* **Parámetros:** Variables que reciben datos del exterior (opcional).
* **Tipo de retorno:** Define el tipo de dato que el método devuelve (opcional).
* **Cuerpo:** Contiene el código que realiza la tarea del método.

**Ejemplo básico:**

```java
public class Calculadora {
    // Método para sumar dos números
    public int sumar(int num1, int num2) {
        return num1 + num2;
    }
}
```

**Explicación:**

* **`public int sumar(int num1, int num2)`:**
    * **`public`**: Especifica que el método es accesible desde cualquier parte del código.
    * **`int`**: Define el tipo de dato que el método devuelve (un entero).
    * **`sumar`**: Es el nombre del método.
    * **`(int num1, int num2)`**:  Son los parámetros, que reciben dos enteros como entrada.
* **`return num1 + num2;`**:  El código dentro del cuerpo del método suma los dos parámetros y devuelve el resultado.

**Utilización del método:**

```java
public class Main {
    public static void main(String[] args) {
        Calculadora calculadora = new Calculadora();
        int resultado = calculadora.sumar(5, 3);
        System.out.println("El resultado de la suma es: " + resultado); // Imprime: 8
    }
}
```

**Tipos de métodos:**

* **Métodos de instancia:** Operan sobre objetos específicos de la clase.
* **Métodos estáticos:** No se asocian a objetos y se invocan directamente usando el nombre de la clase.
* **Métodos abstractos:**  Definen solo la firma (nombre, parámetros y tipo de retorno) y su implementación se realiza en clases hijas.
* **Métodos finales:** Impiden que los métodos sean redefinidos en clases derivadas.

**Ejemplos:**

#### **Método de instancia:**

```java
public class Persona {
    private String nombre;
    private int edad;

    public void mostrarInformacion() {
        System.out.println("Nombre: " + nombre);
        System.out.println("Edad: " + edad);
    }
}
```

####  **Método estático:**

```java
public class Utilidades {
    public static int calcularAreaRectangulo(int base, int altura) {
        return base * altura;
    }
}
```

####  **Método abstracto:**

```java
public abstract class Animal {
    public abstract void hacerSonido();
}
```

####  **Método final:**

```java
public class Circulo {
    private double radio;

    public final double calcularArea() {
        return Math.PI * radio * radio;
    }
}
```

## Métodos Getter y Setter: Acceso Controlado a Atributos en POO Java

Los métodos getter y setter son métodos especiales utilizados en la programación orientada a objetos (POO) de Java para proporcionar un **acceso controlado** a los atributos (campos) de una clase.  

**¿Por qué usar getters y setters?**

* **Encapsulación:** Implementa el principio de encapsulamiento, ocultando los detalles internos de la clase y exponiendo una interfaz pública para interactuar con los atributos.
* **Validación de datos:** Permite validar los datos antes de asignarlos a los atributos, asegurando la integridad de los datos.
* **Flexibilidad:** Permite modificar la implementación interna de la clase sin afectar el código que la utiliza.
* **Control de acceso:** Permite controlar cómo se manipulan los atributos, por ejemplo, con restricciones de lectura o escritura.

**Ejemplo:**

```java
public class Persona {
    private String nombre;
    private int edad;

    // Getter para el atributo nombre
    public String getNombre() {
        return nombre;
    }

    // Setter para el atributo nombre
    public void setNombre(String nombre) {
        this.nombre = nombre;
    }

    // Getter para el atributo edad
    public int getEdad() {
        return edad;
    }

    // Setter para el atributo edad
    public void setEdad(int edad) {
        if (edad >= 0) { // Validación de la edad
            this.edad = edad;
        } else {
            System.out.println("La edad debe ser un valor positivo.");
        }
    }
}
```

**Explicación:**

* **`getNombre()`:** Getter para el atributo `nombre`. Simplemente devuelve el valor del atributo `nombre`.
* **`setNombre(String nombre)`:** Setter para el atributo `nombre`. Asigna el valor del parámetro `nombre` al atributo `nombre` de la clase.
* **`getEdad()`:** Getter para el atributo `edad`. Devuelve el valor del atributo `edad`.
* **`setEdad(int edad)`:** Setter para el atributo `edad`. Valida la edad antes de asignarla al atributo. Si la edad es negativa, muestra un mensaje de error.

**Uso de getters y setters:**

```java
public class Main {
    public static void main(String[] args) {
        Persona persona = new Persona();
        persona.setNombre("Juan");
        persona.setEdad(30);

        System.out.println("Nombre: " + persona.getNombre());
        System.out.println("Edad: " + persona.getEdad());

        persona.setEdad(-10); // Intento de asignar una edad negativa
        System.out.println("Edad: " + persona.getEdad()); // La edad se mantiene como 30
    }
}
```

**Beneficios de usar getters y setters:**

* **Encapsulamiento:** Los atributos se mantienen privados y solo se pueden acceder a través de los getters y setters.
* **Validación:** Se puede aplicar lógica de validación dentro de los setters para garantizar la integridad de los datos.
* **Flexibilidad:** Se puede modificar la implementación de los getters y setters sin afectar al código que los utiliza.

## Encapsulamiento en POO Java: Protegiendo tus datos

El encapsulamiento es uno de los pilares fundamentales de la programación orientada a objetos (POO) en Java. Se trata de una técnica que consiste en **ocultar los detalles internos de una clase y exponer únicamente una interfaz pública para interactuar con ella.**

**¿Por qué es importante el encapsulamiento?**

* **Seguridad:** Protege los datos internos de la clase de modificaciones no autorizadas, asegurando la integridad de la información.
* **Mantenimiento:** Simplifica el mantenimiento del código al reducir el impacto de los cambios en la estructura interna de las clases.
* **Reutilización:** Facilita la reutilización de código al abstraer la implementación interna.

**Ejemplo:**

```java
public class Persona {
    // Atributos (campos) privados
    private String nombre;
    private int edad;

    // Métodos públicos (getters y setters)
    public String getNombre() {
        return nombre;
    }

    public void setNombre(String nombre) {
        this.nombre = nombre;
    }

    public int getEdad() {
        return edad;
    }

    public void setEdad(int edad) {
        this.edad = edad;
    }
}
```

**Explicación:**

1. **Atributos privados:**  Los atributos `nombre` y `edad` se declaran como `private`, lo que significa que solo pueden ser accedidos desde la propia clase `Persona`.
2. **Métodos públicos (getters y setters):** Los métodos `getNombre()`, `setNombre()`, `getEdad()` y `setEdad()` son públicos y proporcionan una interfaz para acceder y modificar los atributos privados.

**Uso del encapsulamiento:**

```java
public class Main {
    public static void main(String[] args) {
        Persona persona = new Persona();

        // Establecer nombre y edad usando los setters
        persona.setNombre("Juan");
        persona.setEdad(30);

        // Obtener nombre y edad usando los getters
        System.out.println("Nombre: " + persona.getNombre());
        System.out.println("Edad: " + persona.getEdad());
    }
}
```

## Modificadores de acceso en POO Java: Controlando la visibilidad

Los modificadores de acceso en POO Java determinan la **visibilidad** de los miembros de una clase, como atributos (campos) y métodos. Controlan qué partes del código pueden acceder a estos miembros. 

**Tipos de modificadores de acceso:**

1. **`public`:** 
   * Visibilidad **global**: Accesible desde cualquier parte del código.
   * Usado para miembros que deben ser accesibles desde cualquier clase.

2. **`protected`:**
   * Visibilidad **dentro del paquete y en subclases:** Accesible desde la misma clase, otras clases dentro del mismo paquete y desde subclases (clases que heredan de la clase actual).
   * Usado para miembros que deben ser accesibles para las subclases o para colaborar con otras clases dentro del mismo paquete.

3. **`private`:**
   * Visibilidad **solo dentro de la clase:** Accesible únicamente desde la propia clase en la que se declaran.
   * Usado para encapsular datos y protegerlos de modificaciones externas no autorizadas.

4. **`default` (sin modificador):**
   * Visibilidad **dentro del paquete:** Accesible solo desde otras clases dentro del mismo paquete.
   * Usado cuando no se necesita una visibilidad específica, por ejemplo, en clases que son internas a un paquete.

**Ejemplo:**

```java
public class Persona { // Clase pública
    private String nombre; // Atributo privado
    protected int edad; // Atributo protegido

    public void mostrarInformacion() { // Método público
        System.out.println("Nombre: " + nombre);
        System.out.println("Edad: " + edad);
    }

    protected void saludar() { // Método protegido
        System.out.println("Hola, soy " + nombre);
    }
}

class Estudiante extends Persona { // Subclase de Persona
    public void estudiar() {
        System.out.println("Estoy estudiando");
        // Puedo acceder a edad porque es protegido
        System.out.println("Tengo " + edad + " años");
        // No puedo acceder a nombre porque es privado
        // System.out.println("Mi nombre es " + nombre); 
    }
}
```

**Uso de los modificadores de acceso:**

* **`public`:** Se utiliza para crear interfaces públicas que permiten a otras clases interactuar con la clase.
* **`protected`:** Se utiliza para permitir que las subclases hereden y extiendan la funcionalidad de la clase, o para colaborar con otras clases dentro del mismo paquete.
* **`private`:** Se utiliza para encapsular los datos internos de la clase y protegerlos de modificaciones externas.
* **`default`:** Se utiliza para miembros que solo son accesibles dentro del paquete en el que se declaran.

## Constructores en POO Java: Iniciando objetos con estilo

Los constructores son métodos especiales en POO Java que se ejecutan automáticamente cuando se crea un nuevo objeto de una clase. Su principal función es **inicializar los atributos (campos) de un objeto** con valores iniciales.

**Características de los constructores:**

* **Nombre:** El nombre del constructor debe ser exactamente igual al nombre de la clase.
* **Sin tipo de retorno:** Los constructores no tienen tipo de retorno, ni siquiera `void`.
* **Se ejecutan automáticamente:** Se invocan al crear un nuevo objeto de la clase.
* **Pueden recibir parámetros:** Para inicializar los atributos con valores específicos.

**Ejemplo básico:**

```java
public class Persona {
    private String nombre;
    private int edad;

    // Constructor sin parámetros
    public Persona() {
        nombre = "Sin nombre";
        edad = 0;
    }

    // Constructor con parámetros
    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }
}
```

**Explicación:**

* **`public Persona()`:** Constructor sin parámetros. Inicializa el nombre como "Sin nombre" y la edad como 0.
* **`public Persona(String nombre, int edad)`:** Constructor con parámetros. Recibe el nombre y la edad del objeto como argumentos y los asigna a los atributos correspondientes.

**Uso de los constructores:**

```java
public class Main {
    public static void main(String[] args) {
        // Creando un objeto Persona sin parámetros
        Persona persona1 = new Persona();
        System.out.println("Persona 1: " + persona1.nombre + ", " + persona1.edad);

        // Creando un objeto Persona con parámetros
        Persona persona2 = new Persona("Juan", 30);
        System.out.println("Persona 2: " + persona2.nombre + ", " + persona2.edad);
    }
}
```

**Salida:**

```
Persona 1: Sin nombre, 0
Persona 2: Juan, 30
```

**Tipos de constructores:**

* **Constructor por defecto:** Se crea automáticamente si no se define ningún constructor explícitamente. Inicializa los atributos con sus valores por defecto (0 para números, `null` para referencias).
* **Constructor parametrizado:** Permite inicializar los atributos con valores específicos proporcionados al crear el objeto.

## Sobrecarga de Métodos y Constructores en POO Java: Flexibilidad a la hora de programar

La sobrecarga en POO Java consiste en definir **varios métodos o constructores con el mismo nombre** pero con **diferentes parámetros**. Esto permite que una clase tenga métodos o constructores con el mismo nombre pero que se comporten de manera diferente según los datos que reciban.

**Sobrecarga de métodos:**

**Ejemplo:**

```java
public class Calculadora {
    public int sumar(int num1, int num2) {
        return num1 + num2;
    }

    public double sumar(double num1, double num2) {
        return num1 + num2;
    }

    public int sumar(int num1, int num2, int num3) {
        return num1 + num2 + num3;
    }
}
```

**Explicación:**

* La clase `Calculadora` tiene tres métodos llamados `sumar`.
* El primer `sumar` recibe dos enteros como parámetros.
* El segundo `sumar` recibe dos números decimales como parámetros.
* El tercer `sumar` recibe tres enteros como parámetros.

**Uso de la sobrecarga de métodos:**

```java
public class Main {
    public static void main(String[] args) {
        Calculadora calculadora = new Calculadora();
        System.out.println(calculadora.sumar(5, 3)); // Llama al primer sumar
        System.out.println(calculadora.sumar(2.5, 1.7)); // Llama al segundo sumar
        System.out.println(calculadora.sumar(1, 2, 3)); // Llama al tercer sumar
    }
}
```

**Sobrecarga de constructores:**

**Ejemplo:**

```java
public class Persona {
    private String nombre;
    private int edad;

    public Persona() {
        nombre = "Sin nombre";
        edad = 0;
    }

    public Persona(String nombre) {
        this.nombre = nombre;
        edad = 0;
    }

    public Persona(String nombre, int edad) {
        this.nombre = nombre;
        this.edad = edad;
    }
}
```

**Explicación:**

* La clase `Persona` tiene tres constructores.
* El primer constructor no recibe parámetros.
* El segundo constructor recibe el nombre como parámetro.
* El tercer constructor recibe el nombre y la edad como parámetros.

**Uso de la sobrecarga de constructores:**

```java
public class Main {
    public static void main(String[] args) {
        Persona persona1 = new Persona(); // Constructor sin parámetros
        Persona persona2 = new Persona("Juan"); // Constructor con nombre
        Persona persona3 = new Persona("Ana", 25); // Constructor con nombre y edad
    }
}
```

**Beneficios de la sobrecarga:**

* **Flexibilidad:** Permite crear métodos o constructores con diferentes comportamientos según los datos que reciban.
* **Reutilización de código:** Se puede reutilizar el mismo nombre de método o constructor con diferentes conjuntos de parámetros.
* **Claridad del código:** Facilita la lectura y comprensión del código, ya que se utilizan nombres de métodos o constructores coherentes.
